# Clipwright 1.0.0

First public release of Clipwright, a Linux desktop utility for preparing camera
and video files before editing.

## Highlights

- Import individual files, folders, or dragged media.
- Inspect media with `ffprobe`, including duration, resolution, frame rate,
  video codec, audio codec, file size, and thumbnails.
- Flag AAC audio that may need conversion for Linux editing workflows.
- Convert AAC audio to PCM in a `.mov` container while copying video without
  re-encoding.
- Merge split GoPro chapter files losslessly.
- Transcode to H.264, H.265/HEVC, ProRes, DNxHR, or stream-copy outputs.
- Quick stream-copy trims.
- Batch rename files with metadata tokens.
- Run jobs in the background with progress and output-folder access.

## Release Artifacts

- `Clipwright-1.0.0-x86_64.AppImage`: recommended Linux download.
- `clipwright-1.0.0.tar.gz`: Python source distribution.
- `clipwright-1.0.0-py3-none-any.whl`: Python wheel.
- GitHub also provides automatic source archives for this tag.

## Linux Requirements

- Linux x86_64.
- `ffmpeg` and `ffprobe` available on `PATH`.

On Debian/Ubuntu:

```bash
sudo apt install ffmpeg
```

## Reliability And Release Hardening

- Added conflict-safe output naming across convert, merge, transcode, and trim
  jobs.
- Hardened ffmpeg subprocess handling to avoid blocked stderr pipes while
  preserving progress reporting.
- Added focused automated tests for output naming, scanner detection, transcode
  argument generation, and ffmpeg progress/error handling.
- Release CI now runs linting, Python compilation, tests, and dependency audit
  before building artifacts.
- AppImage build inputs are pinned and SHA256-verified.
- GitHub Actions used by the release workflow are pinned by commit SHA.

## Notes

The AppImage bundles Python and PyQt6. It does not currently bundle `ffmpeg` or
`ffprobe`, so those must be installed separately.
